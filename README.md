### :computer_mouse:整体架构:

前端（组件化开发） 《--------------》 后端（小程序云开发）《--------------》后台管理系统CMS

> 传统小程序上线部署：
>
> 服务器、域名、备案、网络防护、负载均衡、监控警告
>
> 云开发:
>
> 不需要搭建服务器、专注业务开发、快速上线和迭代
>
> Serverless <----> 无服务开发 <----> 小程序云开发 

### :cloud:云开发提供的能力：

- 云函数： 在云端运行的代码，微信私有协议天然鉴权
- 云数据库： 既可以在小程序端操作又可以在云函数中操作的JSON数据库(NoSql)
- 云存储：在云端存储文件，可以在云端控制台可视化管理
- 云调用： 基于云函数免鉴权使用小程序开放接口的能力
- HTTP API：使用HTTP API开发者可以在已有的服务器上访问云资源，实现与云开发的互通。

> 定时触发云函数读取数据并写入云数据库，打破云函数读取数据条数的限制、云函数的路由优化tcb-router
>
> 小程序用户信息授权流程、多文件上传云存储异步操作、云数据库的模糊查询、索引管理、权限管理
>
> 云数据库中1队N关系的3种设计方式、云调用实现消息模板推送、云数据库多集合查询、分享功能
